# Release keystore

`release.keystore` here is the app's release signing identity. It's gitignored
on purpose (a signing key must never be committed, especially to a public
repo) - **back this file up somewhere safe** (e.g. a password manager or
private cloud drive). If it's lost, future release builds can no longer be
installed as an "update" over an existing install on any device - you'd have
to uninstall the app first, on every device, every time.

Regenerated by (already run once, values are in `mobile/.env`, not here):

```
keytool -genkeypair -v \
  -keystore release.keystore \
  -alias grocy-release \
  -keyalg RSA -keysize 2048 -validity 10000 \
  -storepass "$RELEASE_KEYSTORE_PASSWORD" \
  -dname "CN=Grocy, OU=Personal, O=Personal, L=Unknown, ST=Unknown, C=IN"
```

Modern `keytool` only supports one password for a PKCS12 keystore (store and
key password are the same) - `../plugins/withReleaseSigning.js` uses
`RELEASE_KEYSTORE_PASSWORD` for both when wiring this into the Android build.
